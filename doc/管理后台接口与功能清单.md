# 管理后台接口与功能清单

本文档基于当前代码实现整理，目标是回答两个问题：

1. 现在管理后台后端实际上已经提供了哪些能力。
2. 如果你的前端现在只有“模型列表”和“登录”，接下来应该优先补齐哪些后台页面与功能。

## 1. 当前后台已具备的后端能力

当前代码里，管理后台相关接口并不只有登录和模型列表，已经存在以下模块：

- 管理员认证
- 管理员资料
- 管理员用户列表
- 模型管理
- 分类管理
- 订单管理
- 客户管理
- 客户积分流水查看

对应路由前缀主要是：

- `/api/v1/admin`
- `/api/v1/admin/models`
- `/api/v1/admin/categories`

除登录接口外，其余后台接口都应携带请求头：

```http
Authorization: Bearer <admin_token>
```

## 2. 你现在最该补齐的后台页面

如果前端目前只有“登录”和“模型列表”，建议按下面优先级补齐：

### P0：必须先补

- 后台首页/工作台
  - 展示关键汇总数据：订单数、支付金额、用户数、积分消耗、模型数、分类数。
  - 当前后端没有现成统计接口，这一块需要新增。

- 管理员资料页
  - 登录后获取当前管理员信息。
  - 用于页面右上角用户信息、权限展示、退出登录。

- 模型管理完整页
  - 你现在只有模型列表，还缺：
  - 模型详情
  - 新增模型
  - 编辑模型
  - 删除模型



- 订单管理页
  - 当前后端已提供订单列表、订单详情。
  - 前端应支持：
  - 按订单号筛选
  - 按状态筛选
  - 按用户筛选
  - 查看交易流水

- 客户管理页
  - 当前后端已提供客户列表、客户详情、积分流水查询。
  - 前端应支持：
  - 客户列表
  - 客户详情
  - 最近订单
  - 最近积分流水
  - 独立积分流水分页页

### P1：建议尽快补

- 管理员账号管理
  - 当前只有管理员列表，没有新增、编辑、禁用、改密。

- 客户状态管理
  - 当前只能查用户，不能禁用/启用用户，也不能改资料。

- 积分人工调整
  - 当前只能查看积分流水，不能手动加减积分。

- 充值套餐管理
  - 当前充值套餐看起来是业务配置，但后台没有维护入口。

- 图片内容管理
  - 现有项目里有部分管理写接口分散在别的控制器，不是统一后台模块。
  - 建议在后台统一做图片审核、删除、分类归档、推荐位维护。

### P2：后续完善

- 角色与权限管理
- 操作日志 / 审计日志
- 仪表盘统计趋势图
- 订单退款/补单/人工处理
- 系统配置管理
- API Key / 第三方模型配置管理

## 3. 当前后台接口总览

## 3.1 管理员认证与资料

### 3.1.1 管理员登录

- 方法：`POST`
- 路径：`/api/v1/admin/login`
- 鉴权：否

请求体：

```json
{
  "username": "admin",
  "password": "123456"
}
```

请求体字段说明：

- `username: string`
  - 管理员用户名
  - 必填
- `password: string`
  - 管理员密码
  - 必填

成功返回：

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "jwt-token",
    "userInfo": {
      "id": 1,
      "username": "admin",
      "nickname": "系统管理员",
      "role": "super_admin"
    }
  }
}
```

返回字段说明：

- `code: number`
  - 业务状态码
  - `200` 表示成功
- `message: string`
  - 提示信息
- `data.token: string`
  - 后台 JWT token
  - 后续请求放到 `Authorization` 请求头
- `data.userInfo.id: number`
  - 管理员 ID
- `data.userInfo.username: string`
  - 管理员用户名
- `data.userInfo.nickname: string`
  - 管理员昵称
- `data.userInfo.role: string`
  - 管理员角色

常见失败返回：

- `400`
  - 用户名或密码为空
- `401`
  - 用户名或密码错误
- `403`
  - 账号已禁用
- `500`
  - 后台 JWT 配置缺失

### 3.1.3 管理员退出登录

- 方法：`POST`
- 路径：`/api/v1/admin/logout`
- 鉴权：是

成功返回：

```json
{
  "code": 200,
  "message": "已退出登录"
}
```

说明：

- 当前实现是服务端记录日志，未做 token 拉黑。
- 前端退出时应主动清空本地 token。

## 4. 当前后台还缺哪些关键后端能力

下面这些不是“前端没接”，而是后端目前确实还没有完整接口，建议按优先级补齐。

### 4.1 仪表盘统计接口

建议新增：

- `GET /api/v1/admin/dashboard/overview`

建议返回：

- 今日订单数
- 今日支付金额
- 总用户数
- 今日新增用户
- 总模型数
- 启用模型数
- 总分类数
- 总图片数
- 今日生成次数
- 今日积分消耗

原因：

- 这是后台首页的基础能力。
- 当前没有任何汇总统计接口，前端首页没法做。

### 4.2 管理员账号管理接口

建议新增：

- 管理员新增
- 管理员编辑
- 管理员禁用/启用
- 管理员重置密码
- 管理员删除

原因：

- 当前只有管理员列表。
- 后台系统实际运营时一定会遇到多管理员协作。

### 4.3 客户写操作接口

建议新增：

- 修改用户状态
- 修改用户资料
- 人工调整积分
- 查看用户完整订单列表
- 查看用户完整生成记录列表

原因：

- 当前客户管理只有查询，没有运营动作。

### 4.4 订单运营接口

建议新增：

- 手动标记支付成功
- 手动补单
- 手动关闭订单
- 发起退款
- 订单备注

原因：

- 当前订单仅支持查看，不支持运营干预。

### 4.5 套餐管理接口

建议新增：

- 充值套餐列表
- 新增套餐
- 编辑套餐
- 上下架套餐
- 删除套餐

原因：

- 当前充值业务已经有订单链路，但后台没有套餐维护入口。

### 4.6 内容管理与审核接口

建议新增：

- 图片列表
- 图片审核状态修改
- 图片推荐位维护
- 图片删除
- 图片分类调整
- 用户生成内容审查

原因：

- 这类 AI 产品后台如果没有内容管理，后续运营和风控都会卡住。

### 4.7 权限与审计

建议新增：

- 角色管理
- 权限点管理
- 管理员操作日志
- 关键数据变更审计日志

原因：
