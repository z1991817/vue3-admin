# V3 Admin Vite 宝塔部署说明

本文档适用于当前项目 `v3-admin-vite`，项目技术栈为 `Vue 3 + Vite`，构建产物为纯前端静态文件，最终部署目录为 `dist/`。

## 1. 部署前确认

先确认这几个项目配置：

- 构建命令：`pnpm build`
- 构建产物目录：`dist`
- 当前默认路由模式：`hash`
- 生产环境接口变量：`.env.production` 中的 `VITE_BASE_URL`
- 静态资源基础路径：`.env.production` 中的 `VITE_PUBLIC_PATH`

当前项目默认配置的重点是：

- `VITE_BASE_URL`：生产环境请求的后端接口地址
- `VITE_PUBLIC_PATH`：前端资源访问前缀

如果你准备把项目直接挂在独立域名根目录，例如 `https://admin.xxx.com/`，建议：

```env
VITE_PUBLIC_PATH = /
```

如果你准备把项目挂在二级目录，例如 `https://xxx.com/admin/`，则改成：

```env
VITE_PUBLIC_PATH = /admin/
```

## 2. 推荐部署方式

这个项目部署到宝塔，优先推荐下面两种方式：

### 方式一：本地打包，上传 `dist`

适合前端项目最常见场景，简单、稳定。

流程：

1. 在本地修改生产环境配置文件 `.env.production`
2. 本地执行 `pnpm build`
3. 将生成的 `dist` 目录上传到宝塔站点根目录
4. 在宝塔中把站点设置为静态站点或 Nginx 站点
5. 配置 HTTPS、域名、伪静态

### 方式二：服务器拉代码后构建

适合你后面想持续更新、通过 Git 发布。

流程：

1. 在宝塔服务器安装 `Node.js` 和 `pnpm`
2. 拉取项目代码到站点目录或独立发布目录
3. 配置 `.env.production`
4. 执行 `pnpm install`
5. 执行 `pnpm build`
6. 用 Nginx 指向构建后的 `dist`

如果你只是先把项目上线，建议先用“方式一”。

## 3. 宝塔部署详细步骤

以下按“本地打包上传 `dist`”说明。

### 3.1 修改生产环境配置

编辑项目根目录下的 `.env.production`。

独立域名部署示例：

```env
VITE_BASE_URL = https://api.xxx.com/api/v1
VITE_PUBLIC_PATH = /
```

二级目录部署示例：

```env
VITE_BASE_URL = https://api.xxx.com/api/v1
VITE_PUBLIC_PATH = /admin/
```

注意：

- `VITE_BASE_URL` 必须改成你自己的后端地址
- 如果后端没有开放跨域，前端直接请求会失败
- 如果前后端不在同一域名下，后端需要正确配置 CORS

## 3.2 本地打包

在项目根目录执行：

```bash
pnpm install
pnpm build
```

打包成功后，会生成：

```bash
dist/
```

## 3.3 在宝塔创建站点

进入宝塔面板，按下面操作：

1. 打开“网站”
2. 点击“添加站点”
3. 填写你的域名，例如 `admin.xxx.com`
4. PHP 版本选择“纯静态”或不使用 PHP
5. 创建完成后进入站点目录

站点目录示例：

```bash
/www/wwwroot/admin.xxx.com
```

## 3.4 上传前端构建文件

将本地 `dist` 目录中的所有文件上传到宝塔站点根目录。

最终目录结构应类似：

```bash
/www/wwwroot/admin.xxx.com/index.html
/www/wwwroot/admin.xxx.com/assets
```

注意不是上传整个 `dist` 文件夹嵌套一层，而是把 `dist` 里面的内容放到站点根目录。

## 3.5 配置 Nginx

如果当前项目保持默认 `hash` 路由，通常不需要额外处理刷新 404。

但如果你后面把路由改成 `html5 history` 模式，则必须在宝塔站点的 Nginx 配置里加回退规则。

宝塔站点配置示例：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

适用场景：

- Vue Router 使用 `history` 模式
- 刷新页面返回 404
- 直接访问子路由返回 404

当前项目默认是 `hash` 路由，一般访问形式是：

```text
https://admin.xxx.com/#/dashboard
```

这种情况下通常不需要加前端路由回退。

## 3.6 配置 HTTPS

在宝塔站点里：

1. 进入对应站点
2. 打开“SSL”
3. 申请 Let’s Encrypt 证书或上传证书
4. 开启“强制 HTTPS”

如果你的接口地址也是 HTTPS，浏览器访问会更稳定，避免混合内容问题。

## 4. 如果要部署到二级目录

例如你要部署到：

```text
https://xxx.com/admin/
```

则必须同时满足：

1. `.env.production` 中设置：

```env
VITE_PUBLIC_PATH = /admin/
```

2. 构建后把产物放到对应目录
3. Nginx 访问路径与目录保持一致

如果这里没配对，通常会出现：

- 页面能打开，但样式丢失
- JS 404
- 图片 404

## 5. 接口联调注意事项

当前项目请求地址来自：

```ts
import.meta.env.VITE_BASE_URL
```

也就是生产环境实际请求谁，取决于 `.env.production`。

常见方案：

### 方案一：前端直接请求独立后端域名

示例：

```env
VITE_BASE_URL = https://api.xxx.com/api/v1
```

要求：

- 后端允许当前前端域名跨域
- 后端证书正常

### 方案二：通过 Nginx 反向代理后端接口

示例前端配置：

```env
VITE_BASE_URL = /api/v1
```

对应宝塔 Nginx 配置示例：

```nginx
location /api/v1/ {
    proxy_pass http://127.0.0.1:8080/api/v1/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

这种方式的好处是前端不用处理跨域。

## 6. 服务器直接构建的流程

如果你不想本地打包，可以在服务器上直接构建。

### 6.1 安装运行环境

建议：

- Node.js：`20.19+` 或更高
- pnpm：`10+`

### 6.2 上传源码或拉取代码

把整个项目上传到服务器，例如：

```bash
/www/wwwroot/v3-admin-vite-source
```

### 6.3 安装依赖并构建

进入项目目录执行：

```bash
pnpm install
pnpm build
```

### 6.4 发布构建产物

把 `dist` 目录内容复制到站点根目录，例如：

```bash
/www/wwwroot/admin.xxx.com
```

## 7. 上线检查清单

上线后按下面逐项确认：

1. 首页是否正常打开
2. 浏览器控制台是否有 JS/CSS 404
3. 登录接口是否请求到正确地址
4. 刷新页面是否正常
5. 二级页面直达访问是否正常
6. HTTPS 是否生效
7. 接口是否存在跨域报错

## 8. 常见问题

### 8.1 页面空白

常见原因：

- `VITE_PUBLIC_PATH` 配错
- `assets` 路径加载失败
- 上传目录不对，错误地多套了一层 `dist`

### 8.2 刷新后 404

常见原因：

- 使用了 `history` 路由
- Nginx 没有配置 `try_files`

当前项目默认 `hash` 路由，默认不会因为前端路由刷新而 404。

### 8.3 接口请求失败

常见原因：

- `.env.production` 的 `VITE_BASE_URL` 配错
- 后端没开跨域
- HTTPS 页面请求了 HTTP 接口
- Nginx 反向代理配置错误

### 8.4 样式丢失

常见原因：

- 部署到二级目录，但 `VITE_PUBLIC_PATH` 仍然是 `/`
- 静态资源路径不正确

## 9. 本项目建议的最简上线方案

如果你现在只是要尽快上线，直接按下面做：

1. 把 `.env.production` 改成你自己的接口地址
2. 把 `VITE_PUBLIC_PATH` 改成 `/`
3. 本地执行 `pnpm build`
4. 在宝塔创建一个静态站点
5. 上传 `dist` 内文件到站点根目录
6. 绑定域名并配置 HTTPS
